## Commits Convention

| Commit Type | Description                                                                                                 |
| ----------- | --------------------------------------------------------------------------------------------------------- |
| `feat`      | Adds a new functionality to the project.                                                                  |
| `fix`       | Fixes a bug or issue in the project.                                                                      |
| `docs`      | Changes the project documentation. Ex.: README, code comments.                                            |
| `style`     | Makes changes in appearance, without changing funtionality.                                               |
| `refactor`  | Makes code changes that don't change funtionality.                                                        |
| `test`      | Adds or modifies tests in the project.                                                                    |

##### Commit example

`feat: added new file`
